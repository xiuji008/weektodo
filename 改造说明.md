# WeekToDo 改造说明

## 项目背景

本项目基于开源项目 [WeekToDo](https://github.com/manuelernestog/weektodo) 进行二次开发和定制。WeekToDo 是一款免费开源的简约每周计划应用，专注于隐私保护，支持 Windows、Mac、Linux 和 Web 平台。

**原始项目信息：**
- 作者：Manuel Ernesto Garcia
- 开源协议：GNU GPL 3
- 原始仓库：https://github.com/manuelernestog/weektodo
- 项目版本：v2.2.0

## 改造目的

本次改造旨在为 WeekToDo 添加现代化的 AI 能力、增强用户体验，并优化部署流程，使其更适合国内用户使用。

## 主要改造内容

### 1. AI 功能集成 🤖

#### 1.1 AI 服务配置
- 新增 AI 服务配置模块，支持任何 OpenAI 兼容的 API
- 支持的服务商包括：
  - OpenAI
  - DeepSeek
  - Moonshot (月之暗面)
  - Qwen (通义千问)
  - SiliconFlow
  - 自定义 API 端点

#### 1.2 AI 生成周待办
- 在周视图标题栏新增 "AI 生成周待办" 按钮
- 支持基于用户输入快速生成一周的任务计划
- 智能分析任务优先级和时间分配

#### 1.3 AI 对话功能
- 支持对话上下文管理
- 可配置上下文轮数
- 支持连续对话，提升交互体验

**相关文件：**
- `src/helpers/aiService.js` - AI 服务核心逻辑
- `src/repositories/aiConfigRepository.js` - AI 配置数据管理
- `src/store/modules/main.store.js` - AI 相关状态管理

---

### 2. S3 数据同步功能 ☁️

#### 2.1 新增 S3 同步模块
- 支持任何兼容 S3 协议的存储服务
- 支持的存储服务包括：
  - AWS S3
  - MinIO（私有部署）
  - Cloudflare R2
  - Backblaze B2
  - 其他 S3 兼容服务

#### 2.2 核心功能
- **手动同步**：支持一键上传数据到云端 / 从云端恢复
- **自动同步**：
  - 启动时自动拉取最新数据（pull on start）
  - 数据变化时自动推送（push on change）
- **历史版本管理**：
  - 支持保存数据快照
  - 支持查看历史版本列表
  - 支持从历史版本恢复数据
  - 支持删除旧版本

#### 2.3 安全特性
- API 凭证仅存储在本地设备
- 不会发送到任何第三方服务
- 支持自定义端点 URL（可对接私有存储）

#### 2.4 配置项说明
- **Endpoint URL** - S3 服务端点地址
- **Region** - 存储区域
- **Bucket Name** - 存储桶名称
- **Object Key** - 数据对象键名
- **Access Key ID** - 访问密钥 ID
- **Secret Access Key** - 安全访问密钥
- **Auto Sync** - 启用自动同步

**相关文件：**
- `src/repositories/s3ConfigRepository.js` - S3 配置数据管理（推测）
- `src/store/modules/main.store.js` - S3 同步状态管理
- `src/assets/languages/en.json` - S3 相关翻译（100+ 字符串）

## 如何使用

### 1. 本地开发

```bash
# 克隆项目
git clone https://github.com/你的用户名/weektodo.git
cd weektodo

# 安装依赖
yarn install

# 启动开发服务器
yarn serve        # Web 版本
yarn electron:serve  # 桌面版本
```

### 2. Docker 部署

```bash
# 使用 Docker Compose 启动
docker-compose up -d

# 访问应用
open http://localhost:38080
```

### 3. 配置 AI 功能

1. 打开应用设置
2. 找到 "AI 服务" 配置项
3. 填入 API Endpoint、API Key 和模型名称
4. 保存配置后即可使用 AI 功能

### 4. GitHub Pages 部署

推送代码到 main 分支后，GitHub Actions 会自动构建并部署到 GitHub Pages。

## 新增配置项说明

### AI 服务配置

```json
{
  "aiEndpoint": "https://api.openai.com/v1",
  "aiApiKey": "your-api-key",
  "aiModel": "gpt-3.5-turbo",
  "aiCarryContext": true,
  "aiContextRounds": 10
}
```

### S3 同步配置

```json
{
  "s3Endpoint": "https://s3.amazonaws.com",
  "s3Region": "us-east-1",
  "s3Bucket": "my-weektodo-bucket",
  "s3ObjectKey": "weektodo-data.json",
  "s3AccessKeyId": "your-access-key-id",
  "s3SecretAccessKey": "your-secret-access-key",
  "s3AutoSync": true
}
```

**配置说明：**
- `s3Endpoint` - S3 服务端点（MinIO 示例：`http://localhost:9000`）
- `s3Region` - 存储区域（MinIO 可留空或填 `us-east-1`）
- `s3Bucket` - 存储桶名称（需提前创建）
- `s3ObjectKey` - 数据文件名
- `s3AutoSync` - 是否启用自动同步

### 心情跟踪数据格式

```json
{
  "moodData": {
    "2026-07-02": ["😊", "🎉"],
    "2026-07-03": ["😌"]
  }
}
```
